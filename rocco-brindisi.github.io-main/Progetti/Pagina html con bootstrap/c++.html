La storia

Bjarne Stroustrup iniziò il suo lavoro sul linguaggio nel 1979, motivato dalle sue esperienze durante la realizzazione della tesi di laurea.
Notò che il Simula aveva caratteristiche utili per progetti software di grandi dimensioni, ma era troppo lento per un utilizzo pratico.
Al contrario, il linguaggio BCPL era veloce ma troppo a basso livello per lo sviluppo di applicazioni complesse. Quando Stroustrup fu incaricato di analizzare il kernel di Unix presso i laboratori Bell, decise di arricchire il linguaggio C con alcune caratteristiche di Simula, scegliendo il C per la sua portabilità e velocità.
Inizialmente, con l'aggiunta di funzionalità come classi, classi derivate, controllo rigoroso dei tipi e argomento di default attraverso Cfront, Stroustrup creò "C con classi". La prima versione commerciale fu distribuita nell'ottobre del 1985.
Nel 1983, il nome del linguaggio fu cambiato da "C con classi" a C++, e nuove caratteristiche furono introdotte, come funzioni virtuali, overloading di funzioni, operatori, reference, costanti e controllo dell'utente sulla gestione della memoria.
La prima edizione di "The C++ Programming Language" fu pubblicata nel 1985, fornendo una guida fondamentale nonostante il linguaggio non fosse ancora standardizzato. Nel 1989, la versione 2.0 del C++ introdusse ereditarietà multipla, classi astratte, funzioni membro statiche, funzioni membro const e membri protetti.
Nel 1990, "The Annotated C++ Reference Manual" stabilì le basi per lo standard futuro, incorporando funzionalità come template, eccezioni, namespace, nuovi tipi di cast e il tipo di dato booleano. La libreria standard del C++ ha seguito un'evoluzione parallela al linguaggio stesso. La prima significativa aggiunta fu la libreria degli stream di I/O, seguita dalla Standard Template Library, considerata una delle aggiunte più rilevanti.
Nel 1998, un comitato formato da membri della ANSI e della ISO standardizzò il C++ (ISO/IEC 14882:1998). Nel 2003, il comitato corresse lo standard. Nel 2005, fu pubblicato il "Technical Report 1" (TR1), contenente estensioni alla libreria standard del C++11, sebbene non ufficialmente parte dello standard.
È importante notare che non esiste un proprietario del linguaggio C++, il quale può essere implementato senza l'obbligo di pagare royalty. Tuttavia, il documento di standardizzazione è disponibile solo a pagamento.

Caratteristiche
  • Orientato agli oggetti:
    C++ supporta la programmazione orientata agli oggetti, che consente agli sviluppatori di organizzare il codice in classi e oggetti. Questa caratteristica promuove la modularità, l'incapsulamento e la riutilizzabilità del codice.
  • Compilato:
    Il codice sorgente C++ viene tradotto in linguaggio macchina dal compilatore prima dell'esecuzione del programma. Questo processo di compilazione produce un file eseguibile indipendente dalla piattaforma.
  • Tipizzazione statica:
    C++ è un linguaggio con tipizzazione statica, il che significa che il tipo di dato di una variabile deve essere dichiarato prima dell'uso e non può cambiare durante l'esecuzione del programma.
  • Efficiente:
    C++ è noto per la sua efficienza e velocità di esecuzione. Consente un controllo fine sulla gestione della memoria e offre funzionalità di basso livello che possono essere utilizzate per ottimizzare le prestazioni.
  • Standard Template Library (STL):
    La STL è una parte integrante di C++ e fornisce una raccolta di classi e funzioni template che semplificano e accelerano lo sviluppo del software. Include container, algoritmi e iteratori.
  • Multi-paradigma:
    C++ supporta più paradigmi di programmazione, tra cui programmazione procedurale, programmazione orientata agli oggetti e programmazione generica. Questa flessibilità consente agli sviluppatori di scegliere il paradigma più adatto per una specifica situazione.
  • Gestione della memoria:
    A differenza di molti linguaggi di alto livello che gestiscono automaticamente la memoria, C++ permette agli sviluppatori di gestire manualmente l'allocazione e la deallocazione della memoria. Ciò offre un controllo maggiore sulla gestione delle risorse, ma richiede anche una maggiore attenzione per evitare problemi come memory leaks.
  • Operatori overloading:
    C++ consente agli sviluppatori di definire comportamenti personalizzati per gli operatori, consentendo la creazione di tipi di dati utente che si comportano in modo simile ai tipi di dati nativi.
  • Ricco di funzionalità:
    C++ offre una vasta gamma di funzionalità, tra cui gestione delle eccezioni, manipolazione di stringhe, gestione dei file, supporto per la programmazione di basso livello e altro ancora.
  • Portabilità::
    I programmi C++ possono essere scritti in modo da essere portabili su diverse piattaforme, purché siano rispettati gli standard di codifica e si evitino dipendenze specifiche della piattaforma.
